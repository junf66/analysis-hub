## 1. インフラ前提（最重要）
- kabu APIは**WindowsローカルでkabuステーションアプリがログインState起動中のみ**動く。localhost REST+WebSocket。
- **毎朝の自動ログイン＋`POST /token`（APIPassword→トークン、日次無効化）**を起動シーケンスに組む。
- **スリープ＝全停止**: 取引時間(8:30-15:35)はシステムスリープ禁止（モニタOFFは可）。タスクスケジューラで朝自動起動（スリープ解除タスク）＋Windows Updateのアクティブ時間設定。実弾増額段階では**Windows VPS**（24h稼働・株Botter標準構成）へ移行推奨。watchdog原則=止まったら黙って再開せず通知→安全側（発注しない）。
- `/register` は**最大50銘柄**（PUSH対象）。銘柄入替を管理する。
- 環境: Python 3.11 / stdlibのみ（リポ規約）。`bot/` 配下。**注文実行はdry-runフラグ必須**（デフォルトdry-run・実発注は明示フラグ）。

## 2. 共通基盤（Bot第0号）
- `bot/kabu_client.py`: token取得/失効再取得・REST GET/POST・WebSocket PUSH受信（board/気配）・`/sendorder`/`/cancelorder` ラッパ・`/regulations`（規制照会）・リトライ。
- `bot/signals/`: 夕方バッチ（下記各エッジのシグナル生成。既存 `scripts/` パイプラインを流用し、当日候補を `bot/state/signals_YYYYMMDD.json` に書く）。
- `bot/execute.py`: 朝の気配監視→gap判定→発注→出口の状態機械。
- `bot/log.py`: **全注文・全約定・気配スナップショット・滑り(想定価格vs約定価格)をJSONL記録**（フォワード検証の生データ。これが最重要成果物）。
- ガード（全Bot共通・コードで強制）: 数量上限（1銘柄あたり資金f≤0.25と絶対株数上限の小さい方）・日次損失上限で全停止・dry-run既定。

## 3. エッジ別実行仕様

### ⑫ PEAD（Bot第1号・ロング）
- **夕方**: J-Quants `/td/list`(date=当日) で **DiscTime≥15:00 の決算短信**銘柄リスト生成→翌日の監視対象。
- **翌15:15**: kabu `/ranking?Type=1&ExchangeDivision=ALL`（値上がり率）→ **前日比≥+12%**（許容下限+10%） ∩ 決算リスト。
- **15:25まで**: `/sendorder` **FrontOrderType=16（引成・後場）現物買い**。
- **出口**: +1〜3営業日後の**引成売り**（既定+3日。分割売り可）。
- **ゲート**: TOPIX20日リターン<−3%なら当日スキップ（J-Quants topixで判定）。ザラ場発表(当日DiscTime<15:00)はスルー。
- 記録: 引け値と約定値の差（=引成スリッページ）を必ずログ。

### ④ 増配+来期下方修正ショート
- **夕方**: 既存kouakuパイプライン（`update_all --source kouaku` 相当）で **zouhai_kahou_nx×大引け後** の当日発表分を抽出。
- **翌9:00**: 寄り成行**信用売り**（貸借銘柄のみ。`/regulations`で新規売停止チェック→停止ならスキップ）→ **当日引け買戻**（引成）。一日信用可なら優先。
- ストップは入れない（fade型・正本L688系）。

### ⑩R 小型S高ショート
- **夕方**: 当日**S高引け**検出（kabu `/ranking?Type=1`上位→`/symbol`のUpperLimitと終値一致、またはJ-Quants日足のC==UL）×非プライム小型×**貸借**。
- **朝**: PUSH気配で**gap +5〜10%（中GU）のみ**発注。GD/フラット/大GU(>10%)はスキップ。
- **breadth ゲート**: 当日の全市場S高数>15なら見送り・≤9なら通常（正本の唯一の実用ゲート）。
- **数量: ≤50単元(=5,000株)** かつ極小サイズ（成行空売り51単元規制内）。`/regulations`で売り禁スキップ。
- **出口: 引け買戻のみ（hold-to-close・ストップ/逆指値は絶対入れない**＝早決済は−0.6〜0.8%の実測大損）。

### S安リバウンドロング（🟡候補・極小固定）
- **夕方**: 当日**S安引け**検出（C==LowerLimit）×非プライム小型（**信用含む**・ロングは貸借不要。ただし新規上場・master未収載は除外）。
- **朝**: PUSH気配監視。**indicative gap ≤−12%**（前日終値比）の間だけ**成行買いを板寄せに置く**（気配が−12%より浅くなったら即取消）。
- **10:00までに初値が付かなければ自動取消・見送り**（深GDの44%は後場ずれ＝朝トレ不成立）。
- **出口: 10:30に成行売り**（引けまで持たない＝引けは全部吐き出す）。
- 位置づけ＝**実弾フォワード（滑り実測）**。サイズは最小単元固定。現物の同日1回転はOK（差金決済は再回転がNG）。

### ①B 中型PO GD買い（レア・見逃し厳禁）
- **夕方**: TDnet題名（公募/売出）から当日PO発表を検知 → ScaleCat=Mid400（equities_master）なら翌朝監視。
- **朝**: gap<0 かつ **>−10%（激深≥10%は除外）** なら寄り成行買い → **+1〜10分で成行売り**（スキャル実測: +1分+0.76%/勝82%）または当日引け売り。
- ⑥（受渡日gap<+0.5%・調達≥300億）・⑦（売出のみ決定日寄り売り→引け）・②（REIT・数日）は同じ夕方バッチのPOカレンダーから日付トリガで発注。

### IPO初値×GU20-50% 1分スキャ（🟡研究候補・極小固定）
- **事前**: 上場カレンダー＋**公募価格**を `bot/state/ipo_calendar.json` に登録（公募価格の入力が唯一の人間タッチ。JPX新規上場情報から）。
- **上場日**: 新規銘柄を `/register` → **終日PUSH気配監視**（IPO初値は9:00に限らず午後形成もある）。
- **エントリー**: indicative気配/公募価格−1 が **+20〜50%** の間だけ板寄せ成行買いを生かす（帯を外れたら即取消）→**初値で約定**。
- **出口**: 初値約定検知→**+3分（1〜5分帯）で成行売り**（+10分は勝率52%に劣化＝持たない）。
- 上場直後は信用不可→現物ロング。同日1回転OK。位置づけ＝研究候補の実弾フォワード（滑り実測）・最小単元固定。

## 4. 共通の運用ゲート（全て機械化）
- 日次損失上限超→全Bot停止・通知。
- `/regulations` 照会（売り禁・注意喚起）→ ショート系スキップ。
- ログから週次で: 滑り分布・約定率・エッジ別損益 → 月次で失効モニタ（rolling α。撤退判断は人間＝正本の失効条件表）。

## 5. Done の定義
1. dry-runで1週間、シグナル生成→発注判定→ログまで無エラー。
2. ⑫を最小サイズで実弾、**引成スリッページの実測分布**をレポート（正本の残関門を埋める）。
3. 正本への結果記録（学びは都度・確認なしで `edge_playbook.md` へ）。
