## 2. 再現手順（自己完結・committed データのみ）
```
python -m scripts.edge_candidates.analyze_lockup_short   # 末尾 section_true_terms が上表を出力
```
- 入力(すべて committed): `data/edge_candidates/ipo_lockup_terms.json`(EDINET実条項146社)、
  `data/edge_candidates/ipo_96ut_ratings.json`、`data/edge_candidates/ipo_bars_raw.json`(IPO日足)、
  `data/edge_candidates/topix_daily.json`。
- ロジック: 上場日=IPO日足の最初の足。解除日=上場+89暦日→`_onafter`で営業日化。
  エントリE=その翌営業日。出口X=E+(N-1)営業日。short net = −(E寄→X引の対TOPIX超過) − 0.15。
  クラスタtは解除月(YYYY-MM)でクラスタ。
- EDINET条項の再取得(任意・要 EDINET_API_KEY): `python -m scripts.edge_candidates.fetch_lockup_terms`。
  条項抽出は目論見書「ロックアップについて」節の「後N日目」「起算してN日」「N日間」。例: キオクシア285A=後180日目=2025-06-15。

## 4. 🔴 監査で重点的に潰してほしい論点（弱点候補）
1. **執行の壁（最重要）**: 90日ロック保有48社の現在区分は**信用45/貸借3**。90日マーク=上場3か月で
   ほぼ全部が信用銘柄＝**制度信用で売建不可**。multi-day(+3/+7日)は無期限一般信用の在庫依存(小型は薄/ゼロ)、
   いちにち信用はデイ限定(+1.22%/勝54%)。**統計の本体(multi-day)を確実に出せる執行手段が乏しい**。
   → 「executable な版(デイ+1.22%/勝54%)だけで昇格に値するか」を判定してほしい。
2. **小サンプル**: 90日ロック保有 n46。デイ版は t1.9(非有意)。FDR/DSR 未適用。
3. **多重検定**: 出口(デイ/+3/+5/+7)×条項層別(90有/無/180)×GU帯 で複数試行。DSR/MinBTL での割引が必要。
4. **90日マークの正体**: 「本物の解除」と「一般的な上場後3か月フェード」の合成。後者だけなら別の説明(IPO初値熱の剥落)で
   置換可能か。180日null との整合は取れているが、90日が他の周期効果でないかの確認。
5. **EDINET抽出の取りこぼし**: no_lockup_text 2・no_doc 3・no_name_or_bars 15(=未上場/社名不一致)。
   okサブサンプルにバイアスが無いか(特に古いIPOの取得失敗が分布を歪めないか)。
6. **コスト感度**: short net 0.15%前提。デイ版は損益分岐が薄く、滑り次第で消える可能性。
