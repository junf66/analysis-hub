> 契機: ユーザー指摘「GD−5%超はダメ→実は芯」の類の誤りが他にもないか。統一ルーブリックで全エッジを機械再監査。
> ルーブリック6項目: ①レジーム依存(年次・マニア年除外でt<2?) ②閾値の過剰最適化/誤除外(スイープ) ③PIT/先読み ④データバグ ⑤小n/多重検定 ⑥執行現実性。
> 手法: 各エッジをleave-one-year-out クラスタt で「最も効く年を抜いても t>2 か」を主判定。

## 基盤監査: JPX規模区分(ScaleCat)は妥当か
- **円で切ると連続的に効かない**(GD PO 円3分位 全t<1)。
- **決定打=重複円ゾーン(1000-6000億)でMid/Smallラベルだけで符号反転**: 中型+0.76%/勝69% vs 小型−0.70%/勝44%。
  ＝ScaleCatは円と独立の情報(機関保有/インデックス組入れ/流動性ティア)を持つ実チャネルの代理＝**妥当**(恣意でもラッキーでもない)。
- 留保: ScaleCatは真driver(連続的な機関保有%)の粗い代理・年1回リバランスで陳腐化(境界銘柄=バローHDは2019中型→2020小型)。連続変数があれば改良余地。

## 主要アクション（この監査で判明した"ミス"）
1. **③ 株式分割 announce long = 格下げ**。マニア年(2021/2026)依存で、除外するとt1.5＝非有意。ex-date系と同じレジーム依存。確定T-tierから「レジーム依存の候補」へ落とすべき。
2. **⑦ 売出decide short = 帯が狭すぎ**。確定の"500-1000億"は、実は**300-1500億**が t3.7/n224で上位(500-1000のt2.9/n107より頑健)。GD−5%誤除外と同じ「もっともらしい狭い閾値」の罠。帯を広げるべき。
3. **⑥ = marginal**。条件自体は正当(除外側は有意に負)だが、最良年を抜くとt2.0でギリギリ・n61。過信しない。
4. **①B = sub条件"10%以内GD"はin-sample改善**(t2.6→3.2はn4を後から外した効果)。芯はany GD/t2.6まで。
5. **②④は本物**(全サイクル/最良年除外でt>2.7)。ScaleCat基盤も妥当。

### 不採用44件の false-negative スキャン（＝"誤って棄却したもの"探し）
「統計的に何かあるが棄却/保留」の高リスク項目を再点検した結論＝**台帳は健全・GD型の"データ誤読"は無い**:
- **S安リバウンド 大GD≤−10% long**: +1.07%/t2.35/n522/8-10年+ だが**勝率33%宝くじ・損益分岐0.4%でコスト脆弱・分足で朝10:00ピーク/引けは死**＝正しくcaveat済の「候補」。誤棄却でない。
- **空売り残"水準"高 long**: +20d Q5+0.28%/t2.3だが2024マイナスで年次不安定＝正しく🟡監視どまり。
- **移動平均乖離 深押し≤−15% long**: +7d+0.72%/t2.7だが薄・S安と同族＝正しく保留。
- **ロックアップ解除 short**: 統計的に90日本命だが執行の壁で保留＝理由が執行、データでない。
- **業務提携×小型 short**: 旧+0.89%が独立再計算で+0.3%/勝50%に降格＝正しく降格。
- → いずれも**文書化された正しい理由（執行/コスト/不安定/再現せず）で保留・降格**しており、GD−5%型の「データ誤読による誤棄却」は発見されず。技術/トレンド/台風系は本質的にnullで false-negative リスクなし。
