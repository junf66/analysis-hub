## 毎日のルーティン（3コマ）

### ① 引け後スキャン（15:00〜夜）＝明日の弾込め
当日の開示・S高を見て「明日の候補」をリスト化する。

- [ ] **④ 増配＋来期下方**：適時開示(TDnet/株探「適時開示」)で、**大引け後(15:30以降)** に
      **普通株が「増配（配当予想の上方修正）」と「来期業績の下方修正」を同時発表**していないか。
      - 効きどころ：来期下方が**中程度−30〜−17%**・増配が**大きい(25〜50%)**ほど強い。
      - 該当あり → 明日の「④ショート候補」に記録（銘柄・前日終値）。
- [ ] **⑩R 小型S高**：S高ランキング(株探/マケスピ)で **当日引けS高**の銘柄のうち、
      **非プライム小型(マザーズ/JASDAQ/スタンダード/グロース)×貸借**を抽出。
      - [ ] **市場全体のS高総数(breadth)をカウント** ← 今日の地合いフィルタ：
        - **≤9件＝閑散 → 厚張りOK**（過去 +3.18%/勝率64%）
        - 9〜15件＝中位（+2.69%/勝率61%）
        - **>15件＝過熱 → 薄く or 見送り**（+1.70%/勝率52%・**非有意**・踏み上げ増）
      - 各候補の**前夜の新規売建可否（売り禁でないか）**を楽天で確認 → 売り禁はスキップ。
- [ ] **⑦ 売出PO決定日**：PO関連開示/POカレンダー(株探・トレーダーズ「公募・売出」)で
      **明日が「発行価格決定日」に当たる、売出のみ(希薄化=0)の普通株**がないか。
- [ ] **⑥ PO受渡日**：同カレンダーで **明日が「受渡日」の普通株（調達額≥300億）** がないか。
- [ ] **①B 中型PO・GD**：直近のPO発表で **明日が「発表翌営業日」に当たる TOPIX中型(Mid400)普通株** がないか。
- [ ] **② REIT・事前売り**：当日に **J-REIT が公募増資を発表(×貸借)** したら、**発表翌営業日の寄りで売り**→
      **発行価格決定日の引けまで保有**（＝唯一の数日跨ぎ。オーバーナイト/週末を許容）。季節リピーターREITは除外(正本)。

### ② 寄り前（8:50〜9:00）＝最終判定（gap条件をここで確認）
寄り板/気配で「寄りそうな値」を見て、**gap条件を満たすものだけ**を発注対象に残す。

| ID | 寄り前に満たすべき条件（前日終値比） | 満たさなければ |
|---|---|---|
| ⑩R | **翌寄り +5〜10%（中GU）** で寄りそう | GD/フラット・小GU・大GU(>10%)は**見送り** |
| ①B | **−0.5%以下のGD** で寄りそう | GU・フラットは見送り |
| ⑥ | **gap < +0.5%（GD or フラット）** | **GU(>+0.5%)は見送り** |
| ④ | gap条件なし（増配+来期下方の事実が条件） | — |
| ⑦ | gap条件なし（決定日であることが条件） | — |

- [ ] ⑩R 大GU(>10%)は**特別気配で寄らない/宝くじ**ゆえ除外。中GU(5-10%)は全件寄り付き済で約定可。
- [ ] ショートは改めて**売り禁でないか**最終確認。

### ③ 寄り（9:00）発注 → 引け（15:00）決済
- [ ] 残った候補を **寄り成行**で建てる（S=売り / L=買い）。
- [ ] **当日引けで反対売買**。楽天は寄せ場で出すなら**「大引不成」**（指値を置きつつ大引け成行）で
      15:00決済が確実（「引け」単独は前場引け11:30約定になり得るので注意）。
- [ ] サイズは**全銘柄一律・小口**（薄いエッジは1発でなく多数回で取る）。④⑩Rは勝率高めだが
      **⑩Rは最悪−34.9%の踏み上げテールあり＝極小サイズ厳守**。

## 共通鉄則（貼っておく）
- **予測しない**。条件に合致したら**機械的に全部**入る。合致しなければ**何もしない日が正常**。
- 1日完結＝**オーバーナイトも土日跨ぎも構造上ゼロ**。金曜でも引けで手仕舞うので週末リスクなし。
- ショート(④⑦⑩R)は**楽天信用のみ・売り禁スキップ**。逆日歩はユーザー方針で考慮外。
- **負けてもサイズを縮めない**（薄利を多数回で取る型ゆえ、取り返しの母数を減らさない）。
- ⑩Rだけ地合いフィルタ：**前夜の市場S高 ≤9で厚く・>15で薄く/見送り**。
- X/SNSは見ない（他人の保有スタイルの勝ちは自分のKPIではない）。ブームの上げは**投信側で取る**。

## どこで見るか（情報源）
- **適時開示(④・PO各種)**：TDnet / 株探「決算・適時開示」。引け後15:00〜16:00がピーク。
- **S高ランキング・市場S高総数(⑩R)**：株探「ストップ高」ランキング / 楽天マケスピ。
- **POカレンダー(①B⑥⑦)**：株探・トレーダーズウェブ「公募・売出」。PO発表時に**決定日・受渡日の予定**が載る。
- **規模区分(①B中型/⑩R非プライム小型)**：株探の市場区分・TOPIX規模区分で確認。

> 自動化の余地：この「引け後スキャン」を TDnet/PO/S高 から機械抽出するスクリプト化は可能（要相談）。
> まずは手回しで習慣化し、頻度・勝率が体感と合うか確かめてから自動化を検討する。
